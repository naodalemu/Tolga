Summary: Functions

Core takeaways on organizing code with functions.

  • Functions are blocks of code designed to perform specific tasks, helping to organize code and reduce redundancy.
  • A function has a header (defining its name and arguments) and a body (specifying the operations).
  • Functions are called (or "run") by specifying their name followed by parentheses containing any required arguments.
  • The return statement is used to output a value from a function.
  • Variables within a function have a local scope and cannot be accessed outside the function.
  • Functions can have multiple arguments, no arguments, or even no return statement.
  • The naming convention for functions recommends using lowercase letters with words separated by underscores.
  • Understanding functions is crucial for building efficient and error-free code.
